# ledatic.org

Source for [ledatic.org](https://ledatic.org) — eight hand-rolled static pages served from Cloudflare KV.

## Stack

No React. No Vite. No Next. No Tailwind. No npm. No build step.

HTML + CSS + JS + GLSL. Pages are single files; a shared layer at `_shared/` holds one ~600-line stylesheet, one ~250-line JS runtime, and six raymarched WebGL2 fragment shaders.

## Aesthetic

Neuromancer DNA. HSL 119 99% 46% CRT green on HSL 0 0% 8% charcoal. Sora for display, IBM Plex Mono for labels. Every section eyebrow prefixed `//` like a console log. Cards get hardware-panel chrome: bright LED top-left, dim LED bottom-right, 3D cursor tilt. The logo slash breathes on a 3.2s cycle. Grain + scanlines overlay the whole viewport. Everything is `prefers-reduced-motion` aware.

## Layout

```

rail.html        Rail language

_shared/
  site.css       design tokens + components
  site.js        shader mount, scroll reveals, nav, card tilt, stat rollups, live pollers
  shaders/
    torus.frag       SDF plasma torus (home hero)
    mhd.frag         magnetohydrodynamic raymarching
    mhd_live.frag    live MHD with poll-driven uniforms
    beacon.frag      entropy beacon
    grid.frag        supporting geometry
    dust.frag        volumetric dust

  worker.js        Cloudflare Worker routing KV lookups + auth-gated reports subdomain
  deploy_worker.sh wrangler deploy
deploy.sh          push pages + shared assets to KV (shell-only, no Rail dep)
```

## Deploy

Requires a Cloudflare API token with `Workers KV:Edit` on the correct namespace. Token path defaults to `~/Desktop/rings`; override via `CF_TOKEN`.

```sh
CF_TOKEN=$(cat ~/Desktop/rings) ./deploy.sh              # push all pages + shared assets
CF_TOKEN=$(cat ~/Desktop/rings) ./deploy.sh index.html   # push one file
cd worker && ./deploy_worker.sh                          # push Worker code
```

KV keys are literal paths except `index.html` (mounts at `/`). The Worker reads metadata `{"ct": ...}` on each KV entry to set Content-Type.

## Dynamic pages

A handful of pages (mission control `/system`, `/changelog`, plasma landing details) are generated by the [Rail compiler](https://github.com/zemo-g/rail) in `tools/deploy/gen_*.rail`. Those regenerate on a cron and push to the same KV namespace. The eight pages here are fully hand-authored.
